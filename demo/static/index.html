FlyDesk — Demo Cockpit demo · does not modify flydesk/

Provider-agnostic flight search & booking backend — an interactive demo of the shop → book → ticket flow.

Search flights and book

What runs behind it: the button seeds real Duffel payloads from demo/fixtures/ through the production anti-corruption layer (flydesk/providers/duffel/mapper.py → map_offer) and writes them to Redis under the same key the app uses (flydesk/search/cache.py → offers:<hash>), just with a longer TTL.

Flights currently in the cache (Redis)

The point: offers are ephemeral — they only live for their TTL. This shows exactly what is in Redis right now and when it expires. Read path: demo/seed.py → list_cached() (SCAN over offers:*). Refresh to watch the countdown drop.

Click “Search flights”, then refresh.

Booking history (MongoDB)

What runs behind it: a dump of the durable orders collection in MongoDB (flydesk/common/mongo.py). Bookings are created by the genuine use-case flydesk/bookings/services.py → create_booking (idempotency + outbox). The confirmed → ticketed transition is done asynchronously by the consumer flydesk/events/consumers.py → handle_ticketing (over Kafka) — refresh after a moment to see it change.

How it works — and how to run it

▶️ Run it (locally)

  1. Install Docker Desktop.
  2. From the repo root:
    docker compose -f demo/docker-compose.demo.yml up --build
  3. Open http://localhost:8500 — this demo.
  4. Grafana: http://localhost:3001 (anonymous) → dashboard FlyDesk — Demo.
  5. Prometheus: :9091.

Everything runs offline — no Duffel token, data comes from fixtures. This overlay brings up the whole stack; you don’t run the base docker compose up too.

🧭 2-minute interview walkthrough

  1. Search flights → a list appears (AA / IB / TP from a real capture).
  2. Available flights tab → point at the shrinking TTL = offers are ephemeral.
  3. Book on an offer → a new PNR shows up in Bookings (status confirmed).
  4. Refresh after a few seconds → status flips to ticketed (Kafka consumer).
  5. Book twice (same key) → still one booking = idempotency.
  6. In Grafana the bookings / tickets counters climb live.
